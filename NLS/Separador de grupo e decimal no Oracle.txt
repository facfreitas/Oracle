Existem algumas maneiras de se conseguir isso.
Primeiro você precisa saber qual o escopo da mudança :  apenas para sua sessão ou para todas as sessões do banco de dados.
Caso seja para todas sessões do banco de dados, consulte NLS_DATABASE_PARAMETERS e NLS_INSTANCE_PARAMETERS , se for apenas

para sua sessão consulte NLS_SESSION_PARAMETERS.

Verifique que o separador decimal é ponto e de grupo é virgula.

create table teste (valor number(10,2));

insert into teste values (1000000.99);

commit;

select to_char(valor,'999G999G999D99') from teste

TO_CHAR(VALOR,'999G999G999D99'
------------------------------
   1,000,000.99

1 row selected


Modifique o arquivo de parâmetros, com NLS_NUMERIC_CHARACTERS=',.'
Repita a consulta


select to_char(valor,'999G999G999D99') from teste

TO_CHAR(VALOR,'999G999G999D99'
------------------------------
   1,000,000.99

1 row selected

Existem duas maneiras de corrigir isso.
Na máquina cliente, em caso de sistema operacional windows modifique o registro para a entrada NLS_NUMERIC_CHARACTERS.
No caso de linux, configure uma váriável de ambiente.

Outra opção é o comando alter session:

SQL> alter session set NLS_NUMERIC_CHARACTERS=',.';

Session altered.
